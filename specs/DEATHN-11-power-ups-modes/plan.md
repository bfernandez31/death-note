# Implementation Plan: Power-ups, Game Modes & Main Menu

**Branch**: `DEATHN-11-power-ups-modes` | **Date**: 2026-05-17 | **Spec**: `specs/DEATHN-11-power-ups-modes/spec.md`
**Input**: Feature specification from `specs/DEATHN-11-power-ups-modes/spec.md`

## Summary

Add three power-up types (Freeze, Bomb, Shield) to the typing game, implement a main menu with Survival/Zen/Quit options, add a pause system, create a Zen practice mode with selectable WPM targets, and persist high scores independently per game mode. The core approach is to introduce a `GameScreen` enum to replace boolean state flags, extend the `Zombie` struct with an optional power-up field, and parameterize the existing high score module by `GameMode`.

## Implementation Phases

### Phase 1: Game Screen Enum & Main Menu (Priority: P1)

**Goal**: Replace boolean state management with `GameScreen` enum, implement the main menu, and wire up mode selection.

**Files modified**: `src/main.zig`, `src/zombie_types.zig`

**Changes**:

1. **Add `GameMode` and `GameScreen` enums**
   - `GameMode` in `src/zombie_types.zig:~line 11` (after `ZombieType` enum): `pub const GameMode = enum { survival, zen };`
   - `GameScreen` in `src/main.zig:~line 188` (before `Zombie` struct): `const GameScreen = enum { main_menu, wpm_select, playing, paused, game_over };`
   - Add module-level state: `var current_screen: GameScreen = .main_menu;`, `var game_mode: zt.GameMode = .survival;`, `var menu_selection: u8 = 0;`, `var pause_selection: u8 = 0;`
   - Pattern ref: module-level globals at src/main.zig:138-186

2. **Replace `is_game_over` with `current_screen`**
   - `is_game_over = true` → `current_screen = .game_over`
   - `is_game_over = false` (on restart) → `current_screen = .playing`
   - `if (is_game_over)` → `if (current_screen == .game_over)`
   - `if (!is_game_over and ...)` → `if (current_screen == .playing and ...)`
   - Remove `is_game_over` variable declaration
   - Pattern ref: state gating at src/main.zig:287, 358, 365, 400, 440, 515

3. **Implement menu update/draw in `frame()`**
   - Add `updateMenu()` function: handles Up/Down arrow navigation with circular wrap (FR-014), Enter to select
   - Add `drawMenu()` function: draws title, menu items with highlight, best score display
   - Wire into `frame()`: when `current_screen == .main_menu`, call `updateMenu()` + `drawMenu()` instead of gameplay logic
   - Menu items: "SURVIVAL", "ZEN", "QUIT" — constants `MENU_ITEMS` and `MENU_ITEM_COUNT = 3`
   - Pattern ref: draw layering at src/main.zig:440-501 (state-dependent draw branches)

4. **Implement game start from menu**
   - "Survival" → set `game_mode = .survival`, reset all game state, `current_screen = .playing`
   - "Zen" → `current_screen = .wpm_select`
   - "Quit" → call `raylib.CloseWindow()` or set window-should-close flag
   - Reuse existing reset functions: `resetSessionState()`, `resetScoreState()`, `resetMetricsState()`, `resetZombies()`, `resetBoss()`
   - Pattern ref: restart reset at src/main.zig:474-488

**Tests**:
- `GameScreen` enum has exactly 5 variants
- `GameMode` enum has exactly 2 variants
- Menu selection wraps circularly: `(0 - 1 + 3) % 3 == 2`, `(2 + 1) % 3 == 0`
- Game state reset on mode start produces clean initial state

### Phase 3: Power-up System (Priority: P1)

**Goal**: Implement Freeze, Bomb, and Shield power-ups with carrier designation, inventory, and activation.

**Files modified**: `src/main.zig`, `src/zombie_types.zig`

**Changes**:

1. **Add `PowerUpType` enum to `src/zombie_types.zig`**
   - `pub const PowerUpType = enum { freeze, bomb, shield };`
   - Add constant: `pub const POWER_UP_DROP_CHANCE: u8 = 10;` (10% per FR-002)

2. **Extend `Zombie` struct**
   - Add field: `power_up: ?zt.PowerUpType = null`
   - Pattern ref: existing struct at src/main.zig:189-198

3. **Add power-up module-level state**
   - `var held_power_up: ?zt.PowerUpType = null;`
   - `var freeze_timer: f32 = 0.0;`
   - `var shield_active: bool = false;`
   - Constants: `const FREEZE_DURATION: f32 = 3.0;` (ARD-1)

4. **Power-up carrier designation in `spawnZombie()`**
   - After creating the zombie (src/main.zig:782-791), if `game_mode == .survival`:
     - Roll `rng.intRangeAtMost(u8, 0, 99) < POWER_UP_DROP_CHANCE`
     - If true, assign random type: `rng.intRangeAtMost(u8, 0, 2)` → `.freeze`, `.bomb`, `.shield`
   - Pattern ref: zombie creation at src/main.zig:769-793

5. **Power-up pickup on zombie kill**
   - In `updateZombies()` kill block (src/main.zig:647-660), after `wave_kills += 1`:
     - Check `zomb.power_up != null and held_power_up == null`
     - If true: `held_power_up = zomb.power_up` (FR-004: only pick up if slot empty)
   - Pattern ref: kill flow at src/main.zig:647-660

6. **Power-up activation (Space bar)**
   - In `frame()` update phase, before the `GetCharPressed()` loop:
     - `if (IsKeyPressed(KEY_SPACE) and held_power_up != null)` → activate
     - Freeze: `freeze_timer = FREEZE_DURATION`, clear `held_power_up`
     - Bomb: iterate zombies, destroy all active standards (not boss per FR-008), award score for each, clear `held_power_up`
     - Shield: `shield_active = true`, clear `held_power_up`
   - Space key consumed by `IsKeyPressed()` — remove from `GetCharPressed()` consideration
   - Pattern ref: input handling at src/main.zig:291-318

7. **Freeze effect in update**
   - When `freeze_timer > 0`:
     - In `updateZombies()`: skip `zomb.y += zomb.speed` (line 628)
     - In `updateBoss()`: skip `b.y += b.speed` (line 858) — boss IS frozen per edge case spec
     - `freeze_timer -= raylib.GetFrameTime()` each frame (not in paused state)
     - Spawning continues (new zombies appear but don't move)

8. **Shield effect**
   - In `updateZombies()` at `y >= screen_height` check (src/main.zig:630-634):
     - If `shield_active and game_mode == .survival`:
       - Destroy the zombie, clear `shield_active`, do NOT set `is_dying`
       - Consume shield

9. **Power-up carrier visual in `drawZombies()`**
   - After drawing zombie name (src/main.zig:720):
     - If `zomb.power_up != null`: draw glyph above name with pulsing alpha
     - Freeze: `*` in CRT_ACCENT, Bomb: `!` in CRT_ERR, Shield: `+` in CRT_WARN
   - Pattern ref: zombie name draw at src/main.zig:719-720

10. **HUD power-up display**
    - After existing HUD draws (src/main.zig:515-537):
      - If `held_power_up != null`: draw inventory glyph + label (e.g., "[*] FREEZE")
      - If `freeze_timer > 0`: draw freeze countdown timer
    - Pattern ref: HUD draw section at src/main.zig:515-537

11. **Power-up state reset**
    - Add to `resetSessionState()` (src/main.zig:1035-1041): clear `held_power_up`, `freeze_timer`, `shield_active`

**Tests**:
- `PowerUpType` enum has 3 variants
- `POWER_UP_DROP_CHANCE` == 10
- Zombie struct with `power_up = null` defaults correctly
- Bomb score calculation: N zombies × individual scores
- Shield absorb: `shield_active` transitions from true to false
- Freeze timer: 3.0 → decrement → 0.0 clamp
- Space bar with empty inventory: no state change
- Power-up pickup with full inventory: `held_power_up` unchanged

### Phase 4: Zen Mode (Priority: P2)

**Goal**: Implement Zen practice mode with WPM target selection, no game-over, and simplified HUD.

**Files modified**: `src/main.zig`

**Changes**:

1. **WPM selection screen**
   - Add state: `var zen_wpm_selection: u8 = 0;`
   - Add constant: `const ZEN_WPM_TIERS = [_]u32{ 30, 50, 80 };`
   - `updateWpmSelect()`: Up/Down navigation with wrap, Enter to select, Escape to return to menu
   - `drawWpmSelect()`: title "SELECT WPM TARGET", three options with highlight
   - On select: `game_mode = .zen`, store selected WPM, `current_screen = .playing`

2. **Zen mode spawn configuration**
   - Add state: `var zen_target_wpm: u32 = 50;`
   - In spawn logic, when `game_mode == .zen`: use `deriveWaveTiming(zen_target_wpm)` instead of `getWaveConfig(current_wave)`
   - No wave progression — constant spawn rate throughout session
   - No boss spawns (FR-027): guard `spawnBoss()` call with `game_mode == .survival`

3. **Zen mode zombie-at-bottom behavior**
   - In `updateZombies()` at `y >= screen_height` (src/main.zig:630):
     - If `game_mode == .zen`: `allocator.destroy(zomb)`, `slot.* = null`, continue (no `is_dying`)
     - If `game_mode == .survival`: existing behavior (set `is_dying`)
   - Pattern ref: zombie death at src/main.zig:630-634

4. **Zen mode HUD**
   - When `game_mode == .zen`: draw WPM and accuracy only (FR-026)
   - Hide: score, combo, wave counter, power-up inventory
   - Show: current WPM, accuracy, target WPM for reference

5. **Zen mode power-up suppression**
   - In `spawnZombie()`: skip power-up carrier roll when `game_mode == .zen` (FR-011)
   - Inventory HUD slot hidden

6. **Zen mode session end**
   - No game-over screen (no `is_dying` state transition)
   - Player quits via Escape → pause → "Quit to Menu"
   - On quit: check if WPM/accuracy beat zen best score, save if so

**Tests**:
- `ZEN_WPM_TIERS` has 3 entries: 30, 50, 80
- `deriveWaveTiming(50)` produces valid spawn_delay and fall_speed
- Zen WPM selection wraps: `(0 - 1 + 3) % 3 == 2`
- Zen mode spawn config derives from selected WPM, not wave table

### Phase 5: Per-Mode High Scores (Priority: P2)

**Goal**: Persist and display independent high scores for Survival and Zen modes.

**Files modified**: `src/highscore.zig`, `src/main.zig`

**Changes**:

1. **Parameterize highscore.zig by GameMode**
   - Import `zombie_types.zig` for `GameMode` enum
   - Add `fn filename(mode: GameMode) [*:0]const u8` — returns `"highscore.dat"` for survival, `"highscore-zen.dat"` for zen
   - Add `fn webKey(mode: GameMode) []const u8` — returns `"death-note.highscore"` for survival, `"death-note.highscore.zen"` for zen
   - Modify `pub fn load(mode: GameMode) Record` and `pub fn save(mode: GameMode, record: Record) void`
   - Native: pass `filename(mode)` to `std.c.fopen()`
   - Web: interpolate `webKey(mode)` into localStorage JS
   - Backward compatible: survival uses identical file/key as before (FR-030)

2. **Update main.zig call sites**
   - Replace `best_score` with `best_score_survival` and `best_score_zen`
   - Startup load (src/main.zig:587): `best_score_survival = highscore.load(.survival)`, `best_score_zen = highscore.load(.zen)`
   - Game-over save (src/main.zig:387-395): `highscore.save(.survival, best_score_survival)`
   - Zen session end: compare WPM/accuracy, `highscore.save(.zen, best_score_zen)`
   - Menu display: show best score for most recently played mode (FR-015)

3. **Zen high score comparison**
   - Zen records are compared by WPM first, then accuracy as tiebreaker
   - A new zen best is detected when: `wpm > best_score_zen.wpm or (wpm == best_score_zen.wpm and accuracy > best_score_zen.accuracy)`

**Tests**:

- `filename(.zen)` returns `"highscore-zen.dat"`
- `webKey(.survival)` contains `"death-note.highscore"` (exact match, not substring of zen key)
- `webKey(.zen)` contains `"death-note.highscore.zen"`
